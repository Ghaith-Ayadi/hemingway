After completing any unit of work, run the following steps in order:

## 1. Update learnings

Review everything that was just done. Read `plans/learnings.md` and identify any decisions, discoveries, or non-obvious behaviours that emerged from this work and are not already captured.

If there are new learnings, append them to the correct phase section in `plans/learnings.md`. If a new phase was introduced that has no section yet, add one. Keep entries concise and specific — one bullet per learning.

## 2. Update file header comments

For every file that was created or meaningfully changed in this unit of work, ensure the file has an accurate header comment that describes:
- What the file does (one line)
- How it works — the key mechanism or approach (one line)

The comment goes at the very top of the file, before any imports. Use the language's native comment syntax (`//` for JS/JSX, `/* */` for CSS). Update existing comments if the file's behaviour has changed. Do not add comments to files that were not touched.

## 3. Commit

Stage all modified and new files. Write a commit message that follows the convention already used in this repo (`feat:`, `fix:`, `docs:`, etc.). Commit.

## 4. Push

Push to origin.
